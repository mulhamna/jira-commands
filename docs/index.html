Terminal Jira CLI + MCP

Manage Jira faster from terminal with jirac

Open-source Rust tool for issue management, native backlog type changes, native project moves, JQL search, worklog, attachments, bulk actions, TUI mode, and MCP integration.

Get Started For Non-Tech Users

Not affiliated with, endorsed by, or sponsored by Atlassian.

Split master-detail TUI

$ jirac issue list --project CORE
CORE-183  In Progress  Refactor auth error handling
CORE-177  To Do        Add MCP transition confirmation
CORE-171  Done         Improve TUI issue detail renderer

$ jirac issue transition CORE-183 --to Done
✓ CORE-183 moved to Done

Why jirac

Latest Jira API v3

Uses modern endpoints like /search/jql with cursor pagination.

Custom fields supported

Runtime field introspection without manual YAML mapping.

Terminal + TUI + MCP

Works for direct CLI, keyboard-driven TUI, and agent workflows.

Cross-platform

macOS, Linux, and Windows releases with focused CLI binaries for terminal and MCP workflows.

Screenshots

Issue browser, split master-detail layout, and interactive JQL builder — all keyboard-driven.

Quick Guide for Non-Tech Users

  1. 1. Install

    Use the installer that matches your machine. Windows users can use PowerShell, while macOS and Linux can use the shell installer.

    Tooljirac
  2. 2. Login

    Use your Jira URL, email, and API token to connect your account the first time.

    Runjirac auth login
  3. 3. View your tasks

    See issues assigned to you in a simple terminal list.

    Runjirac issue list
  4. 4. Move issue status

    Select an issue and move it to the next status when work changes.

    Runjirac issue transition PROJ-123

Quickstart (Tech Users)

Installation options

curl -sSL https://raw.githubusercontent.com/mulhamna/jira-commands/main/install.sh | bash
powershell -ExecutionPolicy Bypass -Command "& ([scriptblock]::Create((Invoke-WebRequest 'https://raw.githubusercontent.com/mulhamna/jira-commands/main/install.ps1').Content))"
brew tap mulhamna/tap && brew install jira-commands jira-mcp
cargo install jira-commands
winget install mulhamna.jirac
choco install jirac

Pick one. Homebrew can install both jirac and jirac-mcp; Cargo, the install scripts, Winget, and Chocolatey cover the release binaries too. See INSTALL.md for full matrix.

First login

jirac auth login

Config file path: ~/.config/jira/config.toml (permission 600).

jirac auth status

Interactive TUI

Launch with jirac tui for keyboard-driven issue management, including modal-based backlog type changes, direct project moves, single worklog entry via w, and bulk date-range worklogs via b with confirmation.

Keyboard Description
↑ / kMove to the issue above.
↓ / jMove to the issue below.
EnterOpen details for the selected issue.
cCreate a new issue.
eEdit the selected issue.
yChange the selected backlog item type in a modal, without leaving the TUI.
MMove the selected backlog item to another project in a modal, using Jira native move semantics.
tChange issue status (transition).
CChoose which table columns stay visible and persist that preference.
wAdd a single worklog to the issue.
bAdd bulk worklogs across a date range, with confirmation before submit.
vEdit fix versions.
sAssign the issue to a sprint.
uUpload an attachment to the issue.
?Show shortcut help.

MCP Server

Use jirac-mcp to expose Jira tools to MCP clients. Install it with Homebrew, Cargo, or the release installer flow, depending on your environment.

Install and run

brew tap mulhamna/tap && brew install jira-mcp
cargo install jira-mcp
# macOS / Linux: curl -sSL https://raw.githubusercontent.com/mulhamna/jira-commands/main/install.sh | BINARY=jirac-mcp bash
# Windows: powershell -ExecutionPolicy Bypass -Command "& ([scriptblock]::Create((Invoke-WebRequest 'https://raw.githubusercontent.com/mulhamna/jira-commands/main/install.ps1').Content))" -Binary jirac-mcp
jirac-mcp serve --transport stdio

Client config example

{
  "mcpServers": {
    "jira": {
      "command": "jirac-mcp",
      "args": ["serve", "--transport", "stdio"]
    }
  }
}

Contributing

Contributions are welcome for bug fixes, docs improvements, tests, release automation follow-through, and new features.

Quick contribution flow

  1. Fork and clone the repository.
  2. Create a branch for your change.
  3. Implement your update with tests/docs when relevant.
  4. Run quality checks locally.
  5. Open a pull request with a clear description.

Local checks before PR

cargo fmt --all -- --check
cargo clippy --all-targets --all-features -- -D warnings
cargo test --all
cargo build --all

Commit style: use Conventional Commits (for example: feat:, fix:, docs:, refactor:, test:).

Important: check CLAUDE.md, AGENTS.md, and existing project patterns before submitting major changes.

FAQ

Where do I get Jira API token?

From Atlassian account security settings under API tokens.

Is this replacing Jira web UI?

No. It complements Jira UI for terminal-first and automation workflows.

Does it support custom fields?

Yes, custom fields are discovered dynamically per project.
